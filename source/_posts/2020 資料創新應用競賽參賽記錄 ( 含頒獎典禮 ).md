今年三月多，收到一個素未謀面的群組成員的訊息，希望我可以加入他們組隊參加一些資料應用競賽，而由台北市電腦公會主辦，幾個政府部門協辦的 2020 資料創新應用競賽便是我們第一個參與的競賽。

此次參與中，我們一共報名兩個組別，各提一案參與競賽，且均入選初賽，但鑑於時間及人力安排上，我們最終放棄一組，將心思放在國健署社區長者遠距支持服務及國民健康促進組的比賽，且順利擊敗同組 80 多組隊伍提案，奪下第一名金獎。

資料創新應用競賽
---

資料創新應用競賽至今已是第七屆，在今年的競賽中一共分為五個組別，各自由不同的政府部門來負責 : 

1. 文化部資訊處 : 文化部文化資料創新應用組
2. 內政部統計處 : 內政大數據資料應用組
3. 交通部中央氣象局 : 校園氣象多元應用組
4. 經濟部資訊中心 : 經濟部開放資料應用組
5. 衛福部國民健康署 : 社區長者遠距支持服務及國民健康促進組

每個參賽隊伍每個參賽隊伍中最多五人，需指派一人為團隊代表，報名後可參加海選，主辦單位會邀請產、官、學、研組成評審團，根據「適切性」、「創新性」、「市場性」、「可行性」進行評審，入選後進入決賽，參賽隊伍必須要進行專案展示並 Demo 整個完整方案，最終由評審團依照上述方向，加上「完整性」及「產品發表」兩方面進行綜合審查。

提案內容
---

### 簡介 Intro
AIOT-Team 選定以 『社區長者遠距支持服務及國民健康促進組』 為參賽組別，從「社區老人遠距支援服務網絡」以及「健康促進 e 起來」為主軸進行提案設定。

有鑑於政府積極推動長照2.0政策，以照顧日益漸增的台灣年長者，加上移動設備盛行，對於高齡長者的健康監控、移動照護也有非常快速的進展。若以 2020 年預估的長照照護族群約增加 25 萬人，且用於長者追蹤 GPS 移動設備的平均花費約 10000 元/年來計算，針對老年長者 GPS 應用服務的產值每年約可增加 25 億元/年，可見此產業鏈的發展十足可觀。

### 發展背景 Background

現今台灣移動設備之盛行不言可喻，其中可利用 GPS 達成定位的設備也非常多樣化，舉凡各手機業者推出的智能手錶外，也有專門用於 GPS 定位的許多設備 : 

![](https://i.imgur.com/aBeg0vT.png)
![](https://i.imgur.com/NFKxEUC.png)

當然，直接利用幾乎人手一支的智慧手機，搭配 GPS 定位 APP 也能達到類似的功能


![](https://i.imgur.com/DuWgbXF.png)

有了上述的定位設備後，配合政府開放的公開資料便可以掌握高齡者的每日行為模式以便給予客製化的健康建議服務。除了上述收集資料以及可對照的公開資料外，相對應的演算法發展也十分成熟。

首先會遇到的是 GPS 資料以分秒的速度持續累積，適當進行資料清洗，將雜訊清除，將使預測更加準確。 McKinsey & Company 的一位資料科學家 Sohrab Rahimi 利用了 Numpy, Pnadas 以及 Matplotlib 創建了一個[開源專案](https://github.com/sohrabrahimi/GPS-Data-Cleaning)，可以迅速地進行 GPS 資料清洗。

Alessio D.Marra 等人於 2018 年發表的論文 " *Developing a passive GPS tracking system to study long-term travel behavior* " 則提供了一系列的演算法，以分析一個長時間的 GPS 資料，其中包含資料清洗 ( Data Cleaning ) 、行為辨識 ( Activity Identification ) 以及行程分割 ( Trip Segmentation ) 等。

以上種種發展，均支持我們的提案發展，使其更容易、更可能將整體概念落地發展成實用性極高的產品以及服務。


### 產品概念 Product Concept


![](https://i.imgur.com/c1hvJ9I.png)

整體產品概念如上圖，利用移動設備收及高齡者每日的 GPS 資料，透過核心演算法進行清洗以、行程分割及行為分析後，便可為被服務長者建立行為模式，進而對其模式進行診斷 ( 藉此可了解其運動量是否足夠、人際互動是否充足...等等 )，更進一步還可回饋於使用者，主動發送提醒及建議。

最後，我們也提供直覺、易操作的視覺化系統，可以給予被服務長者的家人、社區、里辦公室清楚的分析結果，也能夠更及時的為高齡長者提供適當的幫助及服務。

### 評審建議

現場另有兩名評審教授給予下列簡單兩項建議，筆者認為這些建議非常的實用，對於接下來的各項競賽都會是需要考量的部分 : 

* 從「資料創新應用」來看，『資料』的使用範圍及對象應盡可能的擴大，雖然我們提的是國健署的案子，但是可能可以跟相關的警政、戶政、...等資料交互應用，這樣一來可以創造更大的商品效能。『創新』的部分應該多注重市場性，在進行設計的同時應該盡可能地提出市場調查，這些也可以作為投資方投資參考。最後的『應用』必須要讓整個提案的概念落地發展，並且要能夠具體化整個堤岸概念非常重要。


* 另外，提案設計要從「使用者」及「系統」兩個層面來討論。在「使用者」的面向來看，我們的產品不僅要易於使用，更要想辦法提供誘因，讓長者使用的黏著度提高。長者「利他」的觀念根深蒂固，可以藉由這樣的誘因 ( 例如:提供子女獎學金  )來使其持續使用商品。至於「系統」層面，資料的 In / Out 很重要，此外，不要將系統封閉在自家產品，如果能與國健署系統結合，具有一種 Open API 的概念會很好。

後記
---

第一次跟不同的人一起合作一個(偽)商業專案，還得到不錯的肯定，這是我始料未及的，要整合不同工作的人，然後利用大家的長才分配工作、安排進度，這真的不是一件容易的事情。

當初決定要加入這個隊伍，一來是對提案的內容很有趣，二來是召集人暨整個團隊的隊長其實非常的有組織概念，之前筆者也曾經想要組織類似的團隊，但太希望大家可以共同決定出團隊的方向、決策的內容等等，反而呈現反作用，最後什麼都沒有辦法進行，但在這個團隊中，方向已經明確，專案內容也確定，再來配合著清楚的分工以及安排，反而能夠讓整個團隊有更好、更高的效率，這或許是讓筆者日後想要組織團隊值得借鏡的地方。

另外，此次放棄了內政部內政大數據資料應用組提案 「他不傻瓜我也很聰明」，但在海選過後，其實內政部統計處長有透漏我們的提案是海選中第一名，他認為這個提案就這樣放棄很可惜，因此，最後我們將這個提案跟內政部聯名報名今年的總統杯黑客松，這也算是「因禍得福」 ( 成語是這樣用的嗎 ? )
